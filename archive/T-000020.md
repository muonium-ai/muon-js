## Goal
Reduce global mutex contention by moving script cache state out of ServerState and locking it independently.

## Acceptance Criteria
- [ ] Move script cache map out of `ServerState` into a dedicated shared lock (`Arc<Mutex<...>>`).
- [ ] Route `SCRIPT` and scripting cache access (`EVAL`/`EVALSHA` cache operations and `FUNCTION` cache operations) through the dedicated script-cache lock.
- [ ] Preserve script command semantics (`SCRIPT LOAD/EXISTS/FLUSH`, `EVALSHA` NOSCRIPT behavior).
- [ ] Validate no regressions with unit tests and ticket-board validation.
