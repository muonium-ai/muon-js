## Goal
Reduce EXEC lock hold time by executing queued no-DB commands without acquiring dbs_state lock, while retaining DB lock for DB-touching queued commands.

## Acceptance Criteria
- [ ] In `EXEC` queue processing, execute queued no-DB commands without acquiring `dbs_state` lock.
- [ ] Acquire DB lock lazily only when the first DB-touching queued command is encountered, and reuse it for remaining DB commands in that `EXEC` batch.
- [ ] Preserve command response behavior for `MULTI`/`EXEC` queue execution.
- [ ] `cargo check --bin mini_redis --features mini-redis` succeeds.
- [ ] `cargo test --lib` remains green.
