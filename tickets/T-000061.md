## Goal
Reduce per-script invocation overhead to improve JS scripting throughput.

## Current state (post-optimization)

The ratio metric is `js_rps / lua_rps` — values > 1.0 mean JS is faster.

Baseline hello ratio: 2.07x (JS 2.07x faster than Lua)
After optimization: 3.53x mean (min 2.86x) — a ~70% improvement.

## Changes made
- Added `JS_EVAL_SCRIPT` flag to skip function wrapping for EVAL scripts
- Implemented `eval_script_body()` supporting top-level `return` without wrapper
- Added script statement caching (FNV-1a hash → parsed stmts) to avoid re-parsing
- Added fast paths for string literals and integers in `eval_expr`
- Optimized `set_keys_argv` for empty arrays
- Removed `wrap_eval_script` from EVAL and EVALSHA paths
