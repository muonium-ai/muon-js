## Goal
Reduce per-script setup/teardown cost so trivial scripts like hello match Lua latency

## Current state

Benchmark `hello` (just `return "Hello, scripting!";`) runs 2.69x slower than the equivalent
Lua script on Redis. This measures pure JS engine overhead: context creation, script parsing,
and result marshaling for the simplest possible script.

## Acceptance Criteria
- [ ] `hello` benchmark ratio drops below 2.0x (Lua/JS)
- [ ] `make lua-js-perf-check` passes gate
- [ ] All cargo tests pass, 10/10 integration tests pass

## Investigation areas
- JS context/environment creation cost per EVAL call
- Script parsing overhead (can we cache parsed AST for repeated EVAL?)
- Result value marshaling back to RESP
- Bytecode compilation overhead for trivial scripts

- `cargo test`
- `make lua-js-perf-check` — check hello ratio
