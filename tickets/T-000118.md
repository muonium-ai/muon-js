## Goal
Animate vehicles so they visibly move on the canvas instead of appearing/disappearing as static queue counts change. Three phases of motion:

1. **Approach**: new vehicles enter from the edge of the canvas and drive toward the intersection, decelerating as they reach the back of the queue.
2. **Queued**: vehicles sit stationary in their queue position, bumper-to-bumper.
3. **Discharge**: when the signal turns green, queued vehicles accelerate through the intersection and exit the opposite edge.

The WASM simulation already tracks queue counts per direction. The JS renderer needs to maintain a visual vehicle list that interpolates between simulation snapshots to produce smooth motion.

## Acceptance Criteria
- [ ] Vehicles visibly drive in from the canvas edge toward the intersection
- [ ] Vehicles decelerate and stop at the back of the queue (no teleporting)
- [ ] On green signal, front vehicles accelerate through the intersection and exit
- [ ] Queue count label matches the number of visible queued vehicles
- [ ] Animation is smooth at 60 fps with up to 30 visible vehicles per pane
- [ ] Both nocache and cached panes animate independently
- [ ] Time-warp (1×–1000×) scales animation speed proportionally
- [ ] Grid view (2×2, 3×3) shows simplified movement (vehicles can pop in/out at smaller scales)
- [ ] No visual glitches on signal phase transitions (green→yellow→red)

## Test Plan
- Visual: watch a full signal cycle — vehicles should approach, queue, then discharge
- Visual: set arrival rate to max (600 vpm) — vehicles should stream in continuously
- Visual: set time-warp to 1× — individual vehicle motion should be clearly visible
- Visual: set time-warp to 1000× — animation should be fast but not glitchy
- Performance: Chrome DevTools → 60 fps with 30+ vehicles on screen

## Notes
Implementation approach:
- Maintain a `VisualVehicle[]` array per pane per direction in JS
- Each frame, reconcile visual vehicles with WASM queue counts (add arrivals, remove discharged)
- Interpolate position with simple kinematics: `pos += velocity * dt`
- Velocity: approaching vehicles move at road speed, queued vehicles at 0, discharging vehicles accelerate
- Keep the WASM simulation logic untouched — animation is purely a rendering concern in app.js
