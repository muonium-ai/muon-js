# T-000047: Perfect hash map for parse_command dispatch

## Goal

Replace the O(n) linear scan in `parse_command()` with a hash-based lookup so that
command dispatch is O(1) instead of iterating through 60+ command strings with
`eq_ignore_ascii_case` on every request.

## Current state

- `server.rs` `parse_command()` has a `COMMANDS` array of 60+ `&str` entries.
- Each incoming request calls `.find(|cmd| input.eq_ignore_ascii_case(cmd.as_bytes()))`.
- This is the hot path — every single client request pays this linear cost.

## Proposed approach

1. Build a `HashMap<Vec<u8>, &'static str>` at startup (using `OnceLock` or `lazy_static`)
   with all command names stored as uppercase keys.
2. On each request, convert the input command to uppercase (in-place on a small stack
   buffer), then do a single hash lookup.
3. Alternatively, use the `phf` crate for a compile-time perfect hash map — zero runtime
   initialization cost.

## Acceptance criteria

- `parse_command` uses hash-based lookup instead of linear scan.
- All 66 tests pass. 10/10 integration tests pass.
- No new dependencies if possible (prefer hand-rolled HashMap); `phf` acceptable if simpler.
