## Goal
In the bytecode VM, change `OpCode::Add` and `OpCode::Sub` integer fast paths
to use `i32::checked_add` / `i32::checked_sub` instead of converting to `f64`,
performing FP arithmetic, and calling `number_to_value()` to convert back.

## Background
The current int+int fast path in `src/vm.rs:155`:
```rust
let ai = a.int32().unwrap_or(0);
let bi = b.int32().unwrap_or(0);
stack.push(number_to_value(ctx, ai as f64 + bi as f64));
```
This does i32→f64 conversion, FP add, then f64→i32 check in `number_to_value`.
For the vast majority of cases (small ints, no overflow), `checked_add` is
cheaper and produces an `i32` directly with no FP involved.

Note: `OpCode::Mul` already uses `i64` overflow check — this is consistent.

## Acceptance Criteria
- [ ] `OpCode::Add` int fast path uses `i32::checked_add`, falls back to f64 on overflow
- [ ] `OpCode::Sub` int fast path uses `i32::checked_sub`, falls back to f64 on overflow
- [ ] All tests pass (`make test`)

## Affected Files
- `src/vm.rs` — `OpCode::Add` and `OpCode::Sub` branches
