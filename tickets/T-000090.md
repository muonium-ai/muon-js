## Goal
Create a multi-threaded benchmark script that tests Lua vs JS scripting
performance with 1,000,000 total requests across multiple concurrent connections.
This complements the existing single-connection benchmark by measuring throughput
under realistic concurrent load.

## Background
The existing `bench_scripting.py` uses a single socket connection and runs
~8,000 total iterations. While useful for per-case profiling, it does not
stress-test the server under concurrent load. A multi-threaded benchmark with
1M requests reveals contention, connection handling, and thread-safety behavior.

## Acceptance Criteria
- New script `scripts/bench_scripting_mt.py` with configurable threads/requests
- Supports both Redis+Lua and mini-redis+JS suites
- 1M total requests distributed across N threads (default 8)
- Reports per-case and aggregate RPS
- Integrated into `tools/lua_js_perf_gate.py` or standalone Makefile targets
- Produces comparison report (JS/Lua ratio per case)

## Test Plan
- Run against Redis (Lua) with 1M requests
- Run against mini-redis (JS) with 1M requests
- Compare results with single-threaded baseline
- Verify no errors/crashes under sustained concurrent load
