## Goal
Close the gap on the `set_members` scripting benchmark where Lua is 2.6x faster than muonjs.

## Current state

Benchmark ratio: **0.39x** (js_rps / lua_rps) — Lua is ~2.6x faster.

Script (`10_set_members.js`):
```js
redis.call('DEL', KEYS[0]);
var n = Number(ARGV[0] || 100);
for (var i = 1; i <= n; i += 1) {
  redis.call('SADD', KEYS[0], 'm' + i);
}
var members = redis.call('SMEMBERS', KEYS[0]);
return members.length;
```

This is a loop-heavy script: 100 iterations of `redis.call('SADD', ...)` with string
concatenation (`'m' + i`), plus a final SMEMBERS + `.length` access.

## Investigation areas
- **String concatenation cost**: `'m' + i` creates a new string each iteration — could be
  expensive if it goes through full coercion path
- **Loop overhead**: `for` loop with `var i = 1; i <= n; i += 1` — check if loop body
  evaluation, variable lookup, and increment have avoidable overhead
- **Number() coercion**: `Number(ARGV[0] || 100)` goes through full type coercion
- **SADD fast path**: Already exists but verify it's being hit for single-member calls
- **Property access `.length`**: `members.length` on an array — check overhead
- **Per-iteration redis.call dispatch**: Each call goes through command parsing even with
  fast path — 100 calls adds up
