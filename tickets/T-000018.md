## Goal
Reduce persistence overhead on the write path and snapshot path using batched transactions and lower per-row execution overhead.

## Acceptance Criteria
- [ ] Optimize AOF flush path to reduce per-entry SQL overhead (batch sizing policy and/or statement reuse) while preserving durability mode semantics.
- [ ] Wrap snapshot full-save flow in efficient transaction boundaries and reduce repeated clone/execute overhead for list/set/hash/zset/stream rows.
- [ ] Keep load/snapshot data correctness and restart recovery behavior unchanged.
- [ ] Show measurable improvement in persistence-enabled benchmarks compared to current behavior.

## Notes
