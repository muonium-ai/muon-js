## Goal
Reduce per-request memory churn in RESP read/write paths by reusing buffers and minimizing temporary string/vec allocations.

## Acceptance Criteria
- [ ] Optimize `read_value`/`read_line` to reduce transient `String` creation and repeated parsing allocations.
- [ ] Ensure encoder paths (`write_value_buf`, `write_array_of_blobs_buf`) avoid unnecessary reallocations for common responses.
- [ ] Keep RESP compatibility for all currently supported types.
- [ ] Demonstrate measurable reduction in allocation-heavy benchmark overhead (especially small command loops).

## Notes
- Hotspot references: `src/mini_redis/resp.rs` (`read_line`, blob and array decode paths, encode helpers).
- Test plan:
	- Run mini-redis parity tests and RESP integration tests.
	- Benchmark small commands (`PING`, `GET`, `SET`) at high request counts.

## Progress Log
- 2026-02-20: Implemented RESP allocation-path optimizations in `src/mini_redis/resp.rs`: reused a single line buffer through recursive decode (`read_value_inner` + `read_line_bytes`), removed transient `String` parsing for integer/blob/array lengths via byte parsing (`parse_i64_bytes`), and added encoder preallocation hints (`estimate_encoded_len`, array payload reserve) for `write_value*` and blob-array responses.
- 2026-02-20: Validation: `cargo test --lib` passed (64/64).
- 2026-02-20: Benchmark evidence from persistence-enabled suite indicates small, noisy shifts on command throughput. Baseline `tmp/mini_redis_benchmark_20260220_230602.log` vs post-change `tmp/mini_redis_benchmark_20260220_231133.log` showed overall +0.42% with mixed command-level deltas (`PING` +0.04%, `GET` +0.11%, `SET` -1.13%, `INCR` -0.88%). Additional run variance remains high.
