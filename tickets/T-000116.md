## Goal
Fix two rendering bugs in vehicle queue display:
1. **Vehicles are centered on the road** instead of positioned in their correct lane. On a multi-lane road, vehicles should occupy individual lanes (left-turning vehicles in the left lane, through traffic in centre lanes, right-turning in right lane).
2. **Vehicle dimensions are swapped** — the current code uses a single `VEH_H = 11` for the along-road axis and the VT cross-road-width (42px for car) across the road. In reality, a car's *length* (~42px) should be along the road axis and its *width* (~18px) across the road. N/S queues should draw tall-narrow rects; E/W queues should draw wide-short rects.

## Acceptance Criteria
- [ ] VT array stores `[color, length, width]` per vehicle type (e.g. car: 42 long, 18 wide)
- [ ] N/S queue vehicles are drawn as `width × length` rects (narrow across road, tall along road)
- [ ] E/W queue vehicles are drawn as `length × width` rects (wide along road, narrow across road)
- [ ] Vehicles occupy distinct lane positions across the road width, not centred on the centreline
- [ ] With 3 lanes, vehicles visually fill 3 columns (N/S) or 3 rows (E/W) within the road surface
- [ ] Queue count labels remain visible and unobstructed
- [ ] No visual regression in 1×1 or grid (2×2, 3×3) views

## Test Plan
- Visual: open demo with 3 lanes, verify vehicles stack in separate lanes
- Visual: compare N/S vs E/W queues — vehicles should be oriented correctly (long axis along travel direction)
- Visual: switch to 1 lane — vehicles should centre in the single lane
- Visual: verify grid view mini-intersections still render cleanly

## Notes
Current VT definition uses `cross-road-width-px` which is actually the vehicle length. Rename and add a proper width dimension.
