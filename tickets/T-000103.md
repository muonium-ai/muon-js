## Goal

Produce downloadable pre-built binaries for both MuonCache and MuonJS for
macOS (x86_64 + aarch64), Linux (x86_64 + aarch64), and Windows (x86_64).
Publish them as GitHub Releases with checksums.

## Scope

### Binary targets

| Binary | Description |
|--------|-------------|
| `muoncache` | Embedded key-value cache server |
| `muonjs` | JavaScript runtime CLI |

### Platforms

| OS | Arch | Rust target |
|----|------|-------------|
| macOS | x86_64 | x86_64-apple-darwin |
| macOS | aarch64 | aarch64-apple-darwin |
| Linux | x86_64 | x86_64-unknown-linux-musl |
| Linux | aarch64 | aarch64-unknown-linux-musl |
| Windows | x86_64 | x86_64-pc-windows-msvc |

### GitHub Actions workflow
- Create `.github/workflows/release.yml`
- Trigger on version tag push (`v*`)
- Matrix build across all platform targets
- Use `cross` for Linux cross-compilation
- Strip binaries for smaller size
- Generate SHA256 checksums
- Upload artifacts as GitHub Release assets
- Release naming: `muonjs-v{VERSION}-{target}.tar.gz` (Unix) / `.zip` (Windows)

### Makefile integration
- `make release` — build release binaries for the current platform
- `make release-all` — cross-compile for all targets (requires cross)

### Release artifacts per platform
```
muonjs-v{VERSION}-{target}.tar.gz (or .zip for Windows)
  ├── muonjs (or muonjs.exe)
  ├── muoncache (or muoncache.exe)
  ├── LICENSE
  └── README.md
```

## Acceptance criteria

- [ ] GitHub Actions workflow builds all 5 targets on tag push
- [ ] Release page has downloadable archives for all platforms
- [ ] SHA256 checksums file is attached to release
- [ ] `make release` builds optimized binaries locally
- [ ] Binaries are statically linked on Linux (musl)
- [ ] VERSION file drives the release version
