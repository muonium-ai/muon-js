## Goal
Build the UI overlay: live metrics panel per simulation (ticks/sec, vehicles processed, avg/max wait time, N/S/E/W queue lengths, simulation time horizon); MuonCache metrics panel (cache hits, misses, hit ratio, avg lookup latency, keys stored); interactive controls (vehicle arrival rate, signal cycle duration, lane count, vehicle type distribution, free left turn toggle); time-warp speed slider (1x/10x/100x/1000x) that visually diverges cached vs non-cached throughput

## Acceptance Criteria
- [ ] All metrics update in real-time (< 100ms display lag behind simulation)
- [ ] Cache metrics panel shows hits, misses, ratio, latency, keys — all non-zero after warm-up
- [ ] Changing arrival rate or signal cycle via sliders resets and restarts both simulations
- [ ] Time-warp slider 1x→1000x causes visibly faster simulation; cached side advances further in simulation time
- [ ] At 1000x warp, cached simulation time horizon is visibly ahead of non-cached within 10 seconds
- [ ] Free left turn toggle changes queue behavior on N/S/E/W approaches

## Test Plan
- Manual: open demo, observe metrics updating every render frame
- Manual: drag arrival rate slider to max, observe queue length metrics increase
- Manual: set time-warp to 1000x, after 10s assert cached sim-time > non-cached sim-time
- Manual: toggle free-left-turn, observe left-turn queue change on signal approaches
