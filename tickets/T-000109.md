## Goal
Wire MuonCache into the simulation engine to cache: intersection state, vehicle positions, lane occupancy maps, signal schedules, turn rule tables, movement decisions. Run two simulation instances side-by-side: one with cache disabled, one cache-enabled. Expose cache metrics (hit count, miss count, hit ratio, avg lookup latency, keys stored) to the UI layer

## Acceptance Criteria
- [ ] Cache-enabled simulation produces identical traffic outcomes to cache-disabled (same seed, same ticks)
- [ ] Cache hit ratio >= 90% after 1,000 warm-up ticks in steady-state traffic
- [ ] Cache-enabled simulation runs >= 5x faster than non-cached at 1,000x time warp (ticks/sec)
- [ ] All six cache metric fields (hits, misses, ratio, avg latency, keys stored) are accessible from WASM JS bindings
- [ ] `cargo test` passes correctness and parity tests for both simulation instances

## Test Plan
- Correctness test: run both instances 1,000 ticks from same seed, assert queue lengths match
- Performance test: run 10,000 ticks, assert cached ticks/sec >= 5x non-cached
- Cache metric test: after 1,000 steady-state ticks, assert hit ratio >= 0.90
- Fuzz: vary arrival rate and signal cycle, assert no cache corruption
