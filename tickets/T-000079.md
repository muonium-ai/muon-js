## Goal
Eliminate duplicate keyword/identifier resolution between fast-path and fallback path in eval_value

## Acceptance Criteria
- [x] No duplicate null/undefined/true/false/this checks in fallback path
- [x] No duplicate NaN/Infinity checks in fallback path
- [x] No duplicate identifier binding resolution in fallback path
- [x] No duplicate builtin/globalThis/global property lookup in fallback path
- [x] All existing tests pass

## Root Cause
When the fast path was added (lines 98-157), the original fallback checks for keyword literals, identifier resolution, builtin dispatch, globalThis, NaN, and Infinity were left in place. Since the fast path catches all ASCII identifiers (same character set as `is_identifier`), these fallback checks were unreachable dead code (~40 lines).

## Fix
Removed 6 blocks of dead code from the fallback path: keyword literals (null/undefined/true/false/this), identifier binding resolution, builtin_or_global closure + dispatch, globalThis check, NaN/Infinity checks, and trailing identifier global lookup.
