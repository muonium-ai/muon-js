## Goal
Add \b, \f, \v, \uXXXX, \xHH, \u{...} escape handling in compiler.rs parse_string_literal

## Acceptance Criteria
- [x] \b, \f, \v produce correct code points (0x08, 0x0C, 0x0B)
- [x] \xHH 2-digit hex escapes produce correct characters
- [x] \uHHHH 4-digit Unicode escapes produce correct characters
- [x] \u{...} variable-length Unicode escapes produce correct characters
- [x] All existing tests pass

## Root Cause
`compiler.rs:parse_string_literal` only handled `\n \r \t \\ \' \" \0`. The escapes `\b \f \v \xHH \uHHHH \u{...}` were passed through as literal backslash + char, producing incorrect strings in the bytecode path. The interpreter path (evals.rs) already handled all of these correctly.

## Fix
Added match arms for `\b`, `\f`, `\v`, `\x`, and `\u` (both `\uHHHH` and `\u{...}` forms) to `parse_string_literal` in compiler.rs. Added `string_escape_sequences_bytecode` test covering all new escape types.
