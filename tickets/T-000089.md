## Goal
Run the Lua vs JavaScript scripting benchmark comparison after the performance
optimizations merged in PRs #28-#32 (T-000088, T-000090, T-000093, T-000094).
Compare against the Feb 20 baseline to measure the impact of:
- AtomTable HashMap index (O(1) atom lookup)
- Checked integer arithmetic in VM Add/Sub
- number_to_value round-trip cast optimization
- Array property alpha-skip optimization

## Background
The previous baseline (Feb 20, 2026) showed:
- Overall mean ratio: 1.20x (JS faster on average)
- Critical hotspots: hash_sum (0.15x), set_members (0.25x), bulk_incr (0.19x)
  where Lua is significantly faster due to computational overhead in the JS runtime.
- Strong JS performance on simple ops: hello (2.07x), redis_call (1.88x), incrby (1.87x)

The recent perf PRs target the VM hot path (integer arithmetic, atom lookup,
number conversion). These should primarily help the compute-heavy cases.
