# T-000055: Avoid full Value clone in Shard::get()

## Goal

Ensure the generic `Shard::get()` method (which clones the entire Value enum) is not
used in hot paths. Replace with specific typed accessors where possible.

## Current state

- `Shard::get()` does `self.data.get(key).cloned()` — cloning the full `Value` enum.
- For `Value::List(VecDeque<Arc<[u8]>>)` this clones the entire deque.
- For `Value::Hash(HashMap<...>)` this clones the entire hash map.
- The Db delegation layer already has typed methods (`get_string`, `list_pop`, etc.)
  that avoid this, but `Shard::get()` may still be used in some paths.

## Proposed approach

1. Audit all callers of `Db::get()` / `Shard::get()` to see if they can use typed
   accessors instead.
2. For callers that need the generic get (e.g., TYPE command), consider returning a
   lightweight enum or reference instead of a full clone.
3. If `Db::get()` is only used for TYPE/EXISTS-like commands, the impact is low and
   this ticket can be deprioritized.

## Acceptance criteria

- No hot-path command uses the generic `Shard::get()` that clones full values.
- All 66 tests pass. 10/10 integration tests pass.
