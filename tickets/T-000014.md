## Goal
Reduce command dispatch overhead by replacing `to_upper_ascii`/`String` normalization with byte-level case-insensitive command matching.

## Acceptance Criteria
- [ ] Remove hot-path `to_upper_ascii` allocations for command names in client command dispatch (`handle_client`/`handle_command`) and script subcommands.
- [ ] Implement efficient case-insensitive byte matching that preserves existing command acceptance behavior.
- [ ] Keep unknown-command and arity error messages unchanged.
- [ ] Demonstrate reduced overhead on command-heavy benchmark cases (`PING`/`SET`/`GET` style and scripting `redis_call`).

## Notes
- Hotspot references: `to_upper_ascii` usage in `src/mini_redis/server.rs` and repeated `cmd.as_ref()` branching.
- Test plan:
	- Run unit/integration tests for command semantics.
	- Run benchmark suite before/after and capture per-case deltas.
