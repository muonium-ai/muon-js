## Goal
Reduce contention between pubsub traffic and DB command paths by removing pubsub from the global server mutex.

## Acceptance Criteria
- [ ] Move pub/sub subscription registry out of `ServerState` global lock into a dedicated shared lock.
- [ ] Route `SUBSCRIBE`/`PUBLISH` handlers through the dedicated pubsub lock without touching DB-state lock.
- [ ] Preserve existing pub/sub response semantics and delivery counts.
- [ ] Validate no regressions in unit tests and board integrity.
