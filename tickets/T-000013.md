## Goal
Remove per-command async runtime boundary overhead in `redis.call`/`redis.pcall` by executing command dispatch through a synchronous scripting path.

## Acceptance Criteria
- [ ] Eliminate `async_std::task::block_on(handle_command(...))` from the scripting bridge (`src/mini_redis/server.rs`).
- [ ] Introduce a sync-safe command execution path for script calls that preserves command semantics and error behavior.
- [ ] Keep `redis.call` throw behavior and `redis.pcall` `{ err = ... }` behavior fully compatible.
- [ ] Demonstrate measurable throughput improvement or lower latency variance in script-heavy cases (`redis_call`, `incrby`, and one hash/set case).

## Notes
- Hotspot reference: `redis_call` in `src/mini_redis/server.rs` currently bridges sync JS to async command handler via `block_on`.
- Test plan:
	- Run `cargo test --lib`.
	- Run `make mini-redis-js-scripting-bench` and compare with baseline.
	- Validate scripting tests in `tests/scripting_js_faithful/`.
