# T-000049: Skip now_ms() syscall when no TTLs are set

## Goal

Avoid calling `SystemTime::now()` (a syscall) on every key access when no keys have
TTL/expiry set.

## Current state

- `Shard::is_expired()` calls `now_ms()` which does `SystemTime::now()` — a syscall.
- This is called on every GET, SET, INCR, LPUSH, LPOP, etc. via the shard methods.
- In benchmarks, the vast majority of keys have no TTL, making this a pure waste.
- The `expires` HashMap is checked via `.get(key)` first, but the `now_ms()` call still
  happens even when there is a match in the expires map.

## Proposed approach

1. **Fast path**: In `is_expired()`, check `self.expires.is_empty()` first. If no keys
   have any TTL set, return `false` immediately without calling `now_ms()`.
2. **Alternative**: Check `self.expires.get(key)` first, only call `now_ms()` when a TTL
   entry actually exists for this specific key. (Current code already does this — verify
   and confirm the syscall is only on the `Some` branch.)
3. **Per-request timestamp caching**: Pass a cached timestamp into shard methods so that
   multiple key operations in one request share a single `now_ms()` call.

## Acceptance criteria

- `is_expired()` does not call `now_ms()` when no TTLs exist.
- All 66 tests pass. 10/10 integration tests pass.
- TTL/EXPIRE/PEXPIRE commands still work correctly.

## Test plan

- `bash tests/run_integration.sh`
- Run `make mini-redis-pipelined-benchmark` and compare INCR/SET throughput.
