## Goal
Remove the 64-byte stack buffer limit for property keys in vm.rs to prevent silent data corruption on long property names

## Acceptance Criteria
- [x] Property keys longer than 64 bytes round-trip correctly through GetProp/SetProp
- [x] Property keys longer than 128 bytes use heap allocation and still round-trip
- [x] GetElem also handles long string indices without truncation
- [x] No performance regression for short keys (stack buffer path ≤128 bytes)
- [x] All existing tests pass

## Root Cause
GetProp, SetProp, and GetElem opcodes in `vm.rs` used `let mut buf = [0u8; 64]` stack buffers with `.min(64)` length clamping. Any property key longer than 64 bytes was silently truncated, causing incorrect property lookups and silent data loss.

## Fix
Introduced `KeyBuf` enum (Stack variant for ≤128 bytes, Heap Vec for longer) and `copy_string_bytes()` helper. Replaced all 6 instances of the truncating `[0u8; 64]` pattern in GetProp, SetProp, and GetElem with the new helper. Added `long_property_key_not_truncated` test covering both stack and heap paths.
