## Goal
Reduce global ServerState lock scope by moving DB vector ownership into a dedicated shared lock in the request path.

## Acceptance Criteria
- [ ] Move DB vector ownership out of `ServerState` into dedicated shared lock state (`Arc<Mutex<Vec<Db>>>`).
- [ ] Route request-path DB access (`handle_lrange_fast`, `handle_command`) through dedicated DB lock while preserving command semantics.
- [ ] Keep scripting `redis.call` command execution functional after DB-state extraction.
- [ ] Validate with unit tests and MuonTickets board validation.
