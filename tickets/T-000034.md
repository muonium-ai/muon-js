# T-000034: Cache parsed function bodies (eliminate re-parsing on every call)

## Goal

Eliminate the single biggest performance bottleneck: every function call re-parses the
function body from source text. `call_closure_with_this` (parser.rs:1769) reads the body
string from the heap, calls `strip_comments_checked` + `normalize_line_continuations` +
`split_statements` every time the function is invoked. In a tight loop calling the same
function 1000×, this means 1000× identical parse work.

## Root cause

`create_function()` stores the function body as a raw source string (`__body__`).
`call_closure_with_this()` → `eval_function_body()` → `strip_comments_checked()` →
`normalize_line_continuations()` → `split_statements()` all run on every call.

## Proposed approach

1. **On first call**, parse the function body to a `Vec<String>` of statement strings (after
   comment stripping, normalization, and splitting).
2. **Store the parsed statements** on the function object as a new property (e.g.
   `__parsed_body__`) or in a side-table keyed by the body's string pointer/hash.
3. **On subsequent calls**, skip `strip_comments_checked`, `normalize_line_continuations`,
   and `split_statements` — directly iterate the cached statement list.
4. `eval_function_body` should accept either source or a pre-parsed `&[&str]`.

### Alternative: bytecode compilation cache

A more aggressive variant compiles the function body to the existing bytecode IR in
`compiler.rs` on first call and re-executes via `vm.rs`. This is a larger change but
would give the highest payoff. Can be done as a follow-up (T-000038).

## Acceptance criteria

- [ ] AC1: A function called N times only parses its body once.
- [ ] AC2: `cargo test --features mini-redis` passes (66 tests).
- [ ] AC3: Scripting benchmark `hash_sum` improves by ≥ 30% (currently 0.16x).

## Test plan

- Unit test: define a function, call it 10000× in a loop, verify result.
- Benchmark: `python3 tmp/run_lua_js_3rounds.py` — compare hash_sum, set_members, bulk_incr before/after.
- Regression: run `cargo test` + `bash tests/run_integration.sh`.
