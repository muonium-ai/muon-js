## Goal
Replace `gc_marks: Vec<*mut u8>` with a `HashSet<usize>` (pointer as key) so
the duplicate-pointer check in `gc_mark_value()` is O(1) instead of O(n).

## Background
In `src/context.rs:1625`, `gc_mark_value()` checks
`self.gc_marks.iter().any(|&p| p == ptr)` — a linear scan — on every
pointer-valued JSValue encountered during GC. This makes the mark phase O(n²)
in the number of heap objects. For large heaps this becomes a significant pause.

## Acceptance Criteria
- [ ] `gc_marks` field changed from `Vec<*mut u8>` to `HashSet<usize>` (or equivalent)
- [ ] `gc_clear_marks()` clears the set
- [ ] `gc_mark_value()` uses `.contains()` / `.insert()` instead of linear scan
- [ ] All tests pass (`make test`)
