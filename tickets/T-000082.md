## Goal
Remove dead-code duplicate command handlers in handle_command that are already handled by handle_no_db_command

## Acceptance Criteria
- [x] PING/ECHO/SELECT removed from handle_command
- [x] Scripting eval path (redis.call) routes through handle_no_db_command first
- [x] No duplicate command logic between the two functions
- [x] All existing tests pass

## Root Cause
`handle_command` contained duplicate PING/ECHO/SELECT match arms identical to those in `handle_no_db_command`. The main connection loop always called `handle_no_db_command` first, making the duplicates in `handle_command` dead code on that path. However, the scripting eval path (`redis.call()`) called `handle_command` directly, bypassing `handle_no_db_command`.

## Fix
Removed duplicate PING/ECHO/SELECT branches from `handle_command`. Added `handle_no_db_command` dispatch to the scripting eval path before falling through to `handle_command`, ensuring consistent handling across all entry points.
