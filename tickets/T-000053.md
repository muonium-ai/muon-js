# T-000053: VM string concat with single allocation

## Goal

Reduce string concatenation in the VM from 3 allocations + 3 copies down to 1 allocation
+ 2 copies.

## Current state

- `vm.rs` OpCode::Add (string path) and OpCode::Concat do:
  ```rust
  let lb = ctx.string_bytes(ls).map(|b| b.to_vec());  // alloc 1
  let rb = ctx.string_bytes(rs).map(|b| b.to_vec());  // alloc 2
  let mut out = Vec::with_capacity(lb.len() + rb.len()); // alloc 3
  out.extend_from_slice(&lb);
  out.extend_from_slice(&rb);
  ```
- Three heap allocations and three memcpy operations per string concat.

## Proposed approach

1. Get byte slices from ctx, compute total length, allocate once, copy directly:
   ```rust
   let la = ctx.string_bytes(ls).unwrap_or(b"");
   // Need to handle borrow — copy first slice if needed
   let la_copy = la.to_vec();
   let rb = ctx.string_bytes(rs).unwrap_or(b"");
   let mut out = Vec::with_capacity(la_copy.len() + rb.len());
   out.extend_from_slice(&la_copy);
   out.extend_from_slice(rb);
   ```
   This reduces from 3 allocs to 2 (or 1 if we can hold both slices simultaneously).
2. If `string_bytes` borrows ctx exclusively, restructure to get lengths first, allocate
   the output buffer, then copy each side.

## Acceptance criteria

- String concatenation uses at most 2 allocations (ideally 1 + the result).
- All tests pass. All mquickjs compatibility tests pass.
