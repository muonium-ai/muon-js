## Goal
Move hash/set storage and transfer paths toward shared-byte representations to reduce clone pressure in read/write-heavy workloads.

## Acceptance Criteria
- [ ] Refactor hot hash/set value paths to use shared byte ownership where safe, minimizing `Vec<u8>` cloning across server/store boundaries.
- [ ] Reduce clone-heavy operations in `hash_getall`, set iteration, and persistence snapshot transfer paths.
- [ ] Preserve command semantics and output formats for all `H*`/`S*` operations.
- [ ] Show measurable throughput improvements in `hash_sum` and `set_members` against baseline.

## Notes
- Hotspot references: `Value::Set(HashSet<Vec<u8>>)` / `Value::Hash(HashMap<Vec<u8>, Vec<u8>>)` and related methods in `src/mini_redis/store.rs`; snapshot cloning in `src/mini_redis/persist.rs`.
- Test plan:
	- Run scripting faithful cases for hash/set workloads and compare medians.
	- Run `cargo test --lib` and parity tests for type/error behavior.
