## Goal
Implement labeled statement syntax (label: { break label; }) so test_language.js test_labels() and test_labels2() pass

## Current state

The mquickjs `test_language.js` file has two test functions that use labeled statements:

```javascript
// test_labels()
do x: { break x; } while(0);
if (1) x: { break x; } else x: { break x; }
while (0) x: { break x; };

// test_labels2()
while (1) label: break
for (;;) label: break
for (i = 0; i < 3; i++) label: { if (i > 0) break; }
```

These fail with `SyntaxError` because the parser doesn't recognize `identifier:` as a
label prefix. All other tests in test_language.js pass.

## Acceptance Criteria
- [ ] `label: { break label; }` exits the labeled block
- [ ] `label: break` works as a labeled break in loops
- [ ] `do label: { break label; } while(0)` works
- [ ] `bash tests/run_mquickjs_tests.sh` — test_language.js passes fully
- [ ] All 66 cargo tests pass, 10/10 integration tests pass

## Notes
Labeled statements are part of the ECMAScript spec. Both the tree-walking interpreter
(eval_expr/split_statements in api.rs/evals.rs) and the bytecode compiler (compiler.rs)
may need updates.
