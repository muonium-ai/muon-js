## Goal
Use atom/byte-based property lookup in LoadGlobal/StoreGlobal to remove repeated UTF-8 and String allocations.

## Acceptance Criteria
- [ ] Eliminate per-instruction `String` allocation in `OpCode::LoadGlobal` and `OpCode::StoreGlobal` hot paths.
- [ ] Keep semantics compatible with mquickjs global property access behavior.
- [ ] Add a microbenchmark showing measurable throughput improvement for global-load/store loops.
- [ ] Ensure no regressions in existing VM/compiler tests and sample scripts.
