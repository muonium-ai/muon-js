## Goal
Unary `!` in `parse_unary` (api.rs ArithParser) consumed the `!` from `!==` and `!=`
operators, breaking them. Fix: check that `!` is not followed by `=` before treating
as unary NOT.

## Fix applied
In `ArithParser::parse_unary` (api.rs:9068), added a guard:
```rust
if let Some(b'!') = self.peek() {
    if self.peek_at(1) != Some(b'=') {
        // only treat as unary NOT when not followed by =
        ...
    }
}
```
