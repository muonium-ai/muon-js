## Goal
When the user changes any interactive control — vehicles/min slider, signal cycle slider, lanes slider, time warp select, or grid size select — the simulation animation freezes or stops entirely. The user must manually refresh the page to get it running again.

Root cause is likely that `lab.reset()` is called on slider input events, and the patched `lab.reset` (which wraps the WASM reset to also clear the PaneAnimators) may be losing the binding or the WASM object is being replaced without re-hooking the animation loop. Additionally, some controls call `lab.reset()` while others don't, creating inconsistent behavior.

## Acceptance Criteria
- [ ] Changing vehicles/min slider keeps animation running smoothly
- [ ] Changing signal cycle slider keeps animation running smoothly
- [ ] Changing lanes slider keeps animation running smoothly (animators rebuild for new lane count)
- [ ] Changing time warp select keeps animation running smoothly
- [ ] Changing grid size keeps animation running (grid view uses bar display, 1x1 uses animated vehicles)
- [ ] Changing traffic mode keeps animation running
- [ ] Changing country rules keeps animation running
- [ ] Reset button keeps animation running
- [ ] No console errors after any control change

## Test Plan
- Open demo, let it run for a few seconds
- Drag each slider and verify vehicles keep moving
- Switch time warp from 1000x to 1x and back — animation should adjust speed
- Switch grid from 1x1 to 2x2 and back — animation should restart cleanly
- Open browser console, interact with all controls — zero errors

## Notes
Investigate whether the monkey-patched `lab.reset` (used to clear PaneAnimators) survives all control paths. Consider using an event-based approach or moving animator reset into the frame loop instead of patching the WASM object's method.
