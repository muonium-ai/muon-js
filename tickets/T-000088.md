## Goal
Replace the linear-scan `AtomTable::find()` in `src/context.rs` with a
`HashMap<u64, u32>` (hash of bytes → atom id) so that `intern_string()` runs
in O(1) instead of O(n) where n = number of interned atoms.

## Background
Every property read/write calls `intern_string()` → `AtomTable::find()`, which
iterates over all entries doing byte-by-byte comparison. As the atom table grows
(hundreds of entries in a typical script), every subsequent property operation
gets proportionally slower. This is the single highest-impact bottleneck in the
runtime.

## Acceptance Criteria
- [ ] `AtomTable` has a `HashMap<u64, u32>` (or similar) index field
- [ ] `find()` does a hash lookup instead of linear scan
- [ ] `push()` updates the index
- [ ] All existing tests pass (`make test`)
- [ ] No new dependencies added (use `std::collections::HashMap` with default hasher or FNV)

## Affected Files
- `src/context.rs` — `AtomTable` struct, `find()`, `push()`, `new()`
