## Goal
Fix the pre-existing bug where bare variable assignment `x = 5` throws
`ReferenceError: not defined` instead of creating an implicit global property
(standard non-strict JavaScript behavior).

## Background
`test_basic_features.sh` "Variable assignment" test has been failing on main.
The test executes `x = 5; x` which expects output `5`, but gets
`ReferenceError: not defined`.

In non-strict JavaScript, assigning to an undeclared variable should create a
property on the global object. The runtime incorrectly checks for the variable's
existence before the assignment and throws `ReferenceError` if not found.

## Root Cause
In `src/api/eval_expr.rs`, the simple-assignment path (`split_assignment`) has
a pre-check that throws `ReferenceError` when the identifier is not found in any
scope or on the global object. This blocks the assignment from reaching
`parse_lvalue`, which already handles the fallback to the global object correctly.
