## Goal

Publish the muon-js runtime as a crate on crates.io so other Rust developers
can embed it via `cargo add muon-js`. Includes the JS runtime library and
optionally the MuonCache embedded store as a feature.

## Scope

### Cargo.toml preparation
- Verify/add required crates.io metadata:
  - `name`, `version`, `edition`
  - `description` — concise one-liner
  - `license` (MIT or Apache-2.0 or dual)
  - `repository` — GitHub URL
  - `homepage`, `documentation`
  - `keywords` — e.g. `["javascript", "runtime", "embedded", "quickjs"]`
  - `categories` — e.g. `["embedded", "wasm"]`
  - `readme = "readme.md"`
- Add `exclude` patterns to keep crate small:
  - `vendor/`, `tmp/`, `target/`, `tickets/`, `devdocs/`, `web/`, `work_product/`
  - Benchmark logs and scripts not needed by library consumers
- Ensure `lib` target is properly declared

### Feature flags
- Default: JS runtime only (small footprint)
- `muoncache` feature: include the embedded cache server module

### API surface review
- Ensure public API in `src/lib.rs` is clean and documented
- Add top-level doc comments with usage examples
- Verify `#[doc(hidden)]` on internal-only items

### Release process
- Use `cargo-release` or manual `cargo publish`
- Add `make cargo-publish` Makefile target
- Ensure `cargo package --list` shows only necessary files
- Verify `cargo doc` builds without warnings

### CI integration
- Add a CI step to run `cargo package --verify` on PRs
- Optionally: auto-publish on version tag (coordinate with T-000103)

## Acceptance criteria

- [x] `cargo package` succeeds with clean output
- [x] `cargo doc` builds without warnings
- [x] Crate size is reasonable (< 500 KB compressed) — 187 KB
- [ ] `cargo publish --dry-run` succeeds
- [ ] Published crate can be added to a fresh project: `cargo add muon-js`
- [x] Basic example works: eval a JS expression from Rust
- [x] VERSION file and Cargo.toml version are in sync

## Test plan

1. `cargo package --list` — verify no unnecessary files included
2. `cargo package --verify` — builds from packaged crate
3. `cargo doc --no-deps` — docs build cleanly
4. `cargo publish --dry-run` — simulated publish succeeds
5. After publish: `cargo add muon-js` in a new project, run example
