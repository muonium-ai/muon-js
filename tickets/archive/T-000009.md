## Goal
Remove unnecessary byte cloning across hash/set command handlers and store APIs to improve throughput of `H*`/`S*` heavy scripting operations.

## Acceptance Criteria
- [ ] Refactor command handlers (`HSET`, `HDEL`, `SADD`, `SREM`) and store entrypoints to avoid building intermediate `Vec<Vec<u8>>` where borrowing/slice-based access is sufficient.
- [ ] Refactor read paths (`HGET`, `SMEMBERS`, related store methods) to minimize clone count while preserving reply ownership and wire compatibility.
- [ ] Preserve existing command behavior and wrong-type errors for hash/set operations.
- [ ] Show measurable improvement in JS scripting benchmarks for `hash_sum` and `set_members` relative to current baseline logs.

## Notes
- Current hotspots include `src/mini_redis/server.rs` argument materialization and `src/mini_redis/store.rs` methods that require owned `Vec<u8>` inputs/outputs in tight loops.
- Test plan:
	- Run targeted scripting bench for `tests/scripting_js_faithful/09_hash_sum.js` and `tests/scripting_js_faithful/10_set_members.js`.
	- Run `cargo test --lib` and integration scripting tests.
	- Confirm no regressions in RESP output shape for hash/set commands via existing test scripts.
