## Goal
Build an automated benchmark harness that: (1) runs cached vs non-cached simulation for N ticks at 1x/100x/1000x warp; (2) records ticks/sec, vehicles processed/sec, cache hit ratio, avg lookup latency to devdocs/trafficlab_bench_baseline.json; (3) asserts cached throughput >= 10x non-cached at 1000x warp; (4) adds a Makefile bench-trafficlab target; (5) prints a summary table to stdout for CI visibility

## Acceptance Criteria
- [ ] `make bench-trafficlab` runs without error and prints a table to stdout
- [ ] Benchmark records: ticks/sec (cached), ticks/sec (non-cached), ratio, hit ratio, avg latency, vehicles/sec
- [ ] Results written to `devdocs/trafficlab_bench_baseline.json` in a schema-stable format
- [ ] At 1000x warp for 10,000 ticks: cached throughput >= 10x non-cached (CI assertion fails if not)
- [ ] `cargo test bench_trafficlab_*` passes as a fast unit-style benchmark
- [ ] Bench output includes speedup ratio and cache hit % so it's self-documenting for launch claims

## Test Plan
- Run `make bench-trafficlab` — inspect stdout table for all 6 fields
- Assert ratio field >= 10.0 at 1000x warp
- Inspect `devdocs/trafficlab_bench_baseline.json` — validate JSON schema has all required keys
- Run twice, assert results are deterministic (< 5% variance) from same seed

## Notes
Benchmark baseline should be committed to `devdocs/trafficlab_bench_baseline.json` so regressions are caught in CI.
