## Goal

Deploy the existing `web/demo/` WASM demo to GitHub Pages so users can try
MuonJS and MuonCache directly in the browser without installing anything.

## Background

A browser demo already exists in `web/demo/` with:
- Vite + TypeScript build
- HTML UI with controls and stats grid
- Worker-based architecture for Rust/WASM execution
- WebGPU metrics dashboard

This ticket focuses on making it deployable via GitHub Pages with automated CI.

## Scope

### WASM build pipeline
- Ensure `wasm-pack build` (or `wasm-bindgen`) produces a working WASM package
- Add `make wasm` target to build the WASM artifacts
- Wire WASM output into the `web/demo/` Vite project

### GitHub Pages deployment
- Create `.github/workflows/pages.yml` workflow
- Steps: build WASM → build Vite app → deploy to `gh-pages` branch
- Trigger on push to `main` (or manual dispatch)
- Use `actions/deploy-pages` or push to `gh-pages` branch

### Demo polish
- Update branding from "Muon Mini-Redis" to "MuonCache" (after T-000102)
- Ensure demo works in Chrome, Firefox, Safari
- Add a JS eval playground section (type JS, see output)
- Include version number in the demo footer

### Documentation
- Add link to live demo in `readme.md`
- Add `web/demo/README.md` with local dev instructions

## Acceptance criteria

- [x] `make wasm` builds WASM artifacts successfully
- [x] `web/demo/` builds and runs locally with `npm run dev`
- [x] GitHub Pages workflow deploys on push to main
- [ ] Demo is accessible at `https://muonium-ai.github.io/muon-js/` (pending first deploy)
- [x] Demo works in Chrome and Firefox
- [x] JS eval playground accepts user input and shows output

## Test plan

1. `make wasm` — verify WASM build completes
2. `cd web/demo && npm install && npm run dev` — local dev works
3. `npm run build` — production build succeeds
4. Push to main → verify GitHub Actions deploys to Pages
5. Visit deployed URL → verify demo loads and runs
6. Test JS eval with sample expressions
