# T-000056: VM GetProp without String allocation for key lookup

## Goal

Eliminate the `String` heap allocation in the VM's GetProp opcode when looking up
object properties by string key.

## Current state

- `vm.rs` OpCode::GetProp does:
  ```rust
  let key_str = core::str::from_utf8(key_bytes).unwrap_or("").to_string();
  let val = js_get_property_str(ctx, obj, &key_str);
  ```
- The `.to_string()` allocates a heap String just to pass it as `&str` to the property
  lookup function.
- This runs on every property access in JS scripts.

## Proposed approach

1. Use `core::str::from_utf8(key_bytes).unwrap_or("")` directly as `&str` — it's
   already a valid `&str`, no need for `.to_string()`.
2. If `js_get_property_str` signature requires `&str`, this should just work:
   ```rust
   let key_str = core::str::from_utf8(key_bytes).unwrap_or("");
   let val = js_get_property_str(ctx, obj, key_str);
   ```
3. Apply the same fix to OpCode::SetProp and the fallback path in GetProp/GetElem.

## Acceptance criteria

- GetProp/SetProp do not allocate a String for string key lookups.
- All tests pass. All mquickjs compatibility tests pass.
