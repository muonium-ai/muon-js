## Goal

Make the **IDM Live tab the default** tab when the page loads, and turn the
simulation into a continuously-running **game loop**: when a collision is
detected the current round ends, scores are recorded (see T-000123), and after a
short pause a new round begins automatically.

## Acceptance Criteria

- [ ] IDM Live tab is active on first page load (not Benchmark)
- [ ] On collision detection (`sim.collisionPair` set), the simulation freezes and
      shows a "Round over" overlay for 3 seconds with the round summary
      (time survived, vehicles cleared, max wait)
- [ ] After the 3-second pause a new `IDMIntersection` is constructed and the RAF
      loop continues seamlessly — no page reload required
- [ ] Round number is shown in the overlay (e.g. "Round 3")
- [ ] The collision freeze overlay already drawn on canvas is reused; the
      "Reload to restart" hint is replaced with a countdown "Restarting in Xs"
- [ ] No console errors between rounds

## Implementation plan

1. In `app.js` `initIDM()` IIFE, track `roundNumber` (starts at 1).
2. Each RAF tick: if `sim.collisionPair` is set and `!roundEnding`:
   - set `roundEnding = true`, record `roundEndTime = performance.now()`
   - call `recordRoundScore(sim, roundNumber)` (T-000123 stub)
3. If `roundEnding` and elapsed since `roundEndTime >= 3000`:
   - increment `roundNumber`, construct new `IDMIntersection`, set `roundEnding = false`
4. `_drawStats` in `idm.js` receives `roundNumber` and `roundEnding` info via
   the sim object or a separate draw-options argument so the overlay can show
   "Round N — Restarting in Xs".

## Test plan

- Open page; IDM tab should be active immediately.
- Let simulation run until collision; overlay should appear within one frame.
- After ~3 s a new round starts with fresh vehicles; round counter increments.
- Rapidly spawn vehicles (high VPM) to force many consecutive collisions and
  confirm no memory leaks or RAF duplication across rounds.
