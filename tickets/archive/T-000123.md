## Goal

At the end of each IDM round (collision detected) write the round's key stats to
the **MuonCache WASM instance** already bundled in `trafficlab/pkg/`.  Fetch the
top-5 scores back immediately so the leaderboard can be rendered (T-000124).

Three metrics must be stored per round:

| metric | source on `IDMIntersection` |
|---|---|
| `duration_s` | `_simTimeSec` (elapsed simulation seconds) |
| `cars_crossed` | `throughput` (count of vehicles that completed a turn) |
| `max_wait_s` | `maxWait` — a new field tracking the highest per-vehicle wait so far |

## Acceptance Criteria

- [ ] `IDMIntersection` gains a `maxWait` field, updated in `step()` whenever a
      vehicle's `waiting` time exceeds the current maximum
- [ ] A `WasmMuonCache` instance is initialised once in `initIDM()` (import from
      `./pkg/muon_js.js`; construct with `new WasmMuonCache()`)
- [ ] On round-end the following commands are executed via `cache.exec_batch(json)`:
      ```
      hset  game:<N>  duration_s    <_simTimeSec rounded to 2 dp>
      hset  game:<N>  cars_crossed  <throughput>
      hset  game:<N>  max_wait_s    <maxWait rounded to 2 dp>
      zadd  leaderboard:duration  <_simTimeSec>   "game:<N>"
      zadd  leaderboard:cars      <throughput>    "game:<N>"
      ```
- [ ] After writing, read top-5 runs ordered by `duration_s` descending:
      ```
      zrange leaderboard:duration 0 4  (reversed — highest score first)
      ```
      For each member, fetch `hget game:<id> duration_s`, `cars_crossed`,
      `max_wait_s`.  Return a JS array `[{rank, id, duration_s, cars_crossed,
      max_wait_s}]` for the renderer.
- [ ] A `getLeaderboard(cache)` helper function is exported from `idm.js` (or
      defined in `app.js`) that encapsulates the above read logic.
- [ ] The leaderboard data is passed to `IDMRenderer.drawLeaderboard()` every
      frame after a round ends, and after the new round starts (so the previous
      round's top-5 is visible during active play).
- [ ] If MuonCache WASM is unavailable (fails to init), the simulation continues
      without scores — no uncaught error.

## Implementation plan

1. Add `this.maxWait = 0` to `IDMIntersection` constructor.
2. In `step()`, inside the per-vehicle wait-accumulation block, add:
   `if (v.waiting > this.maxWait) this.maxWait = v.waiting;`
3. In `trafficlab/app.js` `initIDM()`:
   - `import { WasmMuonCache } from './pkg/muon_js.js';`
   - `let cache = null; try { cache = new WasmMuonCache(); } catch(e) { console.warn('MuonCache unavailable', e); }`
4. Create utility `saveRoundScore(cache, roundNumber, sim)` that builds the
   batch array and calls `cache.exec_batch(JSON.stringify(batch))`.
5. Create utility `async getLeaderboard(cache)` (synchronous `exec` calls; no
   actual async needed) that returns the top-5 array described above.
6. Wire both utilities into the round-end logic from T-000122.

## Test plan

- Run two or three rounds.  Open browser DevTools → Application; or add a debug
  button that calls `cache.exec(JSON.stringify({kind:"zrange", key:"leaderboard:duration", start:0, stop:9}))` and logs the result.
- Verify `hget game:1 duration_s` returns the correct value.
- Verify that with MuonCache removed from the bundle the page still loads and
  the simulation runs (graceful degradation path is tested).
