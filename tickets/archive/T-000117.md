## Goal
Replace the plain colored rectangles with recognisable top-down vehicle shapes drawn on the canvas. Each vehicle type should have a distinct silhouette so users can visually distinguish cars from buses, trucks, motorcycles, and auto-rickshaws without relying solely on color.

Approach options (choose one):
- **Canvas path drawings**: hand-coded `ctx.beginPath()` outlines for each vehicle type (zero external assets, smallest bundle)
- **Inline SVG paths**: define SVG path data as constants, render via `ctx.fill(new Path2D(svgPath))` with rotation
- **Sprite sheet**: a single small PNG sprite sheet loaded at startup (most realistic look, slightly larger payload)

## Acceptance Criteria
- [x] Each of the 5 vehicle types has a visually distinct shape (not just a colored rectangle)
- [x] Vehicles rotate correctly: point upward in N queues, downward in S, left in W, right in E
- [x] Vehicle shapes scale correctly in grid view (2×2, 3×3 mini-intersections)
- [x] No external network requests — all assets are inlined or bundled
- [x] WASM binary + assets total remains under 500 KB
- [x] Rendering at 60 fps is maintained (no jank from complex path drawing)

## Test Plan
- Visual: identify each vehicle type by shape alone (without checking legend colors)
- Visual: verify correct rotation in all 4 queue directions
- Visual: check grid view at 3×3 — vehicles should still be distinguishable
- Performance: confirm 60 fps in Chrome DevTools performance panel

## Notes
Canvas path approach is preferred for zero-dependency simplicity. Example shapes:
- Car: rounded rectangle with windshield notch
- Bus: long rectangle with window dots
- Truck: rectangle with smaller cab section
- Motorcycle: narrow oval
- Auto-rickshaw: small rounded triangle/teardrop
