## Goal
Add a reproducible `redis-benchmark` pipelined performance test script to the
repo so that CI (and contributors) can run the same benchmark used to produce
the March 2026 baseline comparison table.

## Acceptance Criteria
- [x] `tests/run_perf_benchmark.sh` exists, is executable, and produces a comparison table
- [x] Script accepts flags: `--mini-port`, `--redis-port`, `--clients`, `--requests`, `--pipeline`, `--tests`, `--no-redis`, `--baseline`, `--max-regression`
- [x] Script builds mini-redis (release) internally, starts it, runs benchmark, and cleans up on exit
- [x] `make perf-benchmark` (mini vs Redis) and `make perf-benchmark-no-redis` (mini only) targets added to `Makefile`
- [x] `PERF_BENCH_*` variables in Makefile are overridable via environment

## Affected Files
- `tests/run_perf_benchmark.sh` (new)
- `Makefile` — variables block + two new targets + `.PHONY` update

## Test Plan
```bash
# mini-redis only (no Redis dependency – suitable for CI without Redis)
make perf-benchmark-no-redis

# full comparison (requires Redis running on :6379)
make redis-run
make perf-benchmark

# quick smoke test with smaller load
./tests/run_perf_benchmark.sh --requests 100000 --no-redis
```

## Benchmark parameters (matching March 2026 baseline)
- `-c 50` clients
- `-n 1,000,000` requests
- `-P 16` pipeline depth
- Tests: GET, SET, INCR, LPUSH, RPUSH, LPOP, RPOP, SADD, HSET

## Notes
This reproduces the methodology used to generate the March 2026 baseline results
(~1.9M ops/s range), which showed mini-redis at parity or faster on 7 of 9 ops.
The `--no-redis` mode is safe for CI environments that do not run Redis.
