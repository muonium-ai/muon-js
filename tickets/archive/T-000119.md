## Goal
Add two visual overlays on each vehicle in the 1x1 pane view:

1. **Vehicle number** — a small sequential number rendered on top of each vehicle so users can track individual vehicles through the system (e.g. #1, #2, #3...). Numbers reset per direction arm and help users follow a specific vehicle from approach to discharge.

2. **Turn-direction arrow** — a small arrow indicator showing which way the vehicle will go after crossing the intersection:
   - **Straight** (through): up arrow `↑` (or forward arrow relative to travel direction)
   - **Left turn**: left-curving arrow `↰` or `←`
   - **Right turn**: right-curving arrow `↱` or `→`

   Turn direction should be assigned deterministically per vehicle (e.g. based on lane assignment: left lane = left turn, right lane = right turn, middle lanes = straight). This matches real-world lane discipline.

## Acceptance Criteria
- [ ] Each vehicle displays a small number (1-based index within its queue arm)
- [ ] Numbers are legible at default zoom — use contrasting color against vehicle body
- [ ] Each vehicle displays a turn-direction arrow (straight, left, or right)
- [ ] Turn direction is assigned by lane: leftmost lane = left turn, rightmost = right turn, middle = straight
- [ ] With 1 lane, all vehicles show straight arrow
- [ ] With 2 lanes, left lane = left turn, right lane = right (or straight)
- [ ] Arrows rotate correctly with vehicle orientation (N/S/E/W)
- [ ] Overlays are visible but don't obscure vehicle shape identity
- [ ] Grid view (2x2, 3x3) does NOT show overlays (too small)

## Test Plan
- Visual: open demo with 3 lanes, verify numbers 1-N on each arm's vehicles
- Visual: verify left-lane vehicles show left arrow, right-lane show right arrow, middle show straight
- Visual: change to 1 lane — all arrows should be straight
- Visual: check all 4 directions — arrows should rotate with the vehicle

## Notes
- Render number and arrow using `ctx.fillText()` on top of the vehicle shape (after `drawVehicleShape`)
- Use a small font (8-9px) with white or bright color for contrast
- Arrow characters: `↑` (straight), `←` (left), `→` (right) — or use canvas path arrows for crispness
- The number is the 1-based visual index in the queue, not a global vehicle ID
