## Goal
In `js_set_property_str()`, when the target is an array, short-circuit the
numeric-index detection and `NaN`/`Infinity` string checks for property names
that start with a letter or underscore.

## Background
`js_set_property_str()` in `src/api/mod.rs:513` does manual byte-by-byte
index parsing on every call when the target is an array. It then checks for
`"NaN"`, `"Infinity"`, and `parse::<f64>()`. If the name starts with an ASCII
letter or `_`, it can never be a numeric index — skip all numeric checks.

## Acceptance Criteria
- [ ] Early exit from numeric-index detection when name starts with `[a-zA-Z_]`
- [ ] All tests pass (`make test`)
- [ ] No behavioral change for actual numeric index strings

## Affected Files
- `src/api/mod.rs` — `js_set_property_str()`
