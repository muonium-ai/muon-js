## Goal
Reduce avoidable overhead in `EVAL`/`EVALSHA` script preparation and cache lookup while keeping SCRIPT command behavior compatible.

## Acceptance Criteria
- [ ] Avoid repeated large temporary allocations in script wrapping/execution (e.g., wrapper construction and argument vector rebuilding) when semantics allow reuse.
- [ ] Improve `EVALSHA` hot-path behavior so cached-script execution does not require unnecessary cloning/materialization.
- [ ] Preserve current SCRIPT subcommand behavior (`LOAD`, `EXISTS`, `FLUSH`) and error semantics.
- [ ] Demonstrate measurable improvement for scripting workloads with frequent eval calls, or document neutral impact if bridge/command costs dominate.

## Notes
- Candidate functions: `eval_script`, `eval_script_sha`, `ScriptRuntime::set_keys_argv` in `src/mini_redis/server.rs`.
- Test plan:
	- Run scripting benchmark suite before/after with focus on eval-heavy scripts.
	- Run `cargo test --lib` and scripting integration tests.
	- Validate SCRIPT command compatibility manually (`SCRIPT LOAD/EXISTS/FLUSH`, `EVALSHA` miss/hit behavior).
