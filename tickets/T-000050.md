# T-000050: Pre-allocate static OK and error RespValue responses

## Goal

Eliminate per-response heap allocations for common static responses like "OK", "PONG",
and frequent error messages.

## Current state

- Commands like SET, SELECT, PING return `RespValue::Simple("OK".to_string())` — a heap
  allocation of 2 bytes on every successful SET.
- Error messages like "WRONGTYPE Operation against a key holding the wrong kind of value"
  are allocated fresh on every error.
- These strings are identical every time.

## Proposed approach

1. Change `RespValue::Simple(String)` to `RespValue::Simple(Cow<'static, str>)` or add a
   `RespValue::StaticSimple(&'static str)` variant.
2. Alternatively, use `Arc<str>` and pre-build common responses as constants.
3. Simplest approach: define `const` or `static` `RespValue` instances for OK, PONG,
   QUEUED, and the WRONGTYPE error. Return clones of these (cheap if using Arc or &'static).

## Acceptance criteria

- Common responses (OK, PONG, QUEUED, WRONGTYPE) do not allocate on each use.
- All 66 tests pass. 10/10 integration tests pass.

## Test plan

- `cargo test --features mini-redis`
- `bash tests/run_integration.sh`
- Run `make mini-redis-pipelined-benchmark` — expect small but measurable SET improvement.
