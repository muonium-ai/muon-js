## Goal
Extend the simulation to a configurable NxN intersection grid (default 3x3). Support up to 10,000 simultaneous vehicles. Add inter-intersection routing so vehicles travel across the grid. Amplify cache benefit at scale: the cache hit ratio should increase significantly vs single-intersection mode. Add grid toggle to the UI controls

## Acceptance Criteria
- [ ] 3x3 grid (9 intersections) renders correctly in both simulation panes
- [ ] Vehicles route between intersections; positions are consistent across consecutive ticks
- [ ] 10,000 simultaneous vehicles run without panic or memory exhaustion
- [ ] Cache hit ratio in 3x3 mode >= hit ratio in single-intersection mode (cache reuse increases at scale)
- [ ] Grid size selector (1x1, 2x2, 3x3) in UI correctly reinitializes both simulations
- [ ] `cargo test` passes grid routing and scale tests

## Test Plan
- Headless test: initialize 3x3 grid, run 1,000 ticks, assert no intersection overlap or orphaned vehicles
- Scale test: spawn 10,000 vehicles, run 500 ticks, assert memory usage < 512MB WASM heap
- Cache ratio test: compare single vs 3x3 hit ratios after 5,000 ticks warm-up

## Notes
This is an advanced feature; blocks on T-000110 (renderer) to render correctly.
