## Goal

Add a second tab "**IDM Live**" alongside the existing "Benchmark" tab. The IDM
tab runs the **Intelligent Driver Model** (Treiber 2000) — a microscopic car-
following physics model — where every vehicle has its own continuous position,
velocity, and acceleration, updated at 60 fps. The result is visually smooth
traffic flow with realistic deceleration, queue formation, and discharge, similar
to https://www.traffic-simulation.de/intersection.html.

The existing benchmark tab is **unchanged**. This is a new additive feature.

## Acceptance Criteria

- [ ] A tab bar at the top of the page switches between "Benchmark" and "IDM Live"
- [ ] IDM Live tab shows a single signalised intersection (no split-pane, no MuonCache comparison)
- [ ] Vehicles have continuous position, velocity, and smooth deceleration toward red light / queued vehicle ahead
- [ ] Vehicle shapes are rendered with correct orientation for N/S/E/W approach arms
- [ ] Multiple lanes supported (1–6, same control as Benchmark tab)
- [ ] Signal cycle control shared with Benchmark tab controls (or own slider)
- [ ] Vehicles entering from off-canvas edge (no pop-in mid-road)
- [ ] Discharged vehicles travel through intersection and exit on the opposite arm with a smooth arc/straight path
- [ ] Animation stays at 60 fps (nowDelta < 20ms) under all warp settings
- [ ] No console errors

## Implementation plan

### 1. IDM physics model (pure JS, no WASM needed)
Each vehicle is an object `{ id, arm, lane, pos, vel, type }` where:
- `pos` = distance from stop line (positive = approaching, negative = past)
- `vel` = current speed in px/sec
- `arm` = 'n'|'s'|'e'|'w'

IDM acceleration formula per tick:
```
a = a_max * [1 - (vel/v0)^4 - (s*(vel, delta_vel) / gap)^2]
s*(v, dv) = s0 + v*T + v*dv / (2*sqrt(a_max*b_comfort))
gap = pos_ahead - pos - vehicle_length
```

Constants (tunable via sliders):
- `v0` = desired speed = 60 px/sec (≈ 50 km/h scaled)
- `a_max` = 2.5 px/sec²
- `b_comfort` = 3.0 px/sec² (comfortable deceleration)
- `T` = 1.5 sec (time headway)
- `s0` = 8 px (minimum gap)

Signal braking: treat stop line as virtual vehicle with gap = pos, vel = 0 when signal is red/yellow.

### 2. Spawning
- Poisson arrivals derived from veh/min slider
- Vehicles spawn at `pos = ROAD_LENGTH + random(0, 20)` (off-canvas)
- `vel` initialised to `v0`

### 3. Rendering
- Each vehicle is a filled rounded rect (no save/rotate — use axis-aligned rects for N/S arms; swap w/h for E/W arms)
- Color by type (same VT palette as Benchmark tab)
- Draw front-most 3 vehicles' number badge only

### 4. Tab switching
- `<div id="tab-bar">` with two buttons: "Benchmark" and "IDM Live"
- Toggle a CSS class on `#sim-canvas` wrapper or use two canvases
- Active tab's render loop runs; inactive tab pauses its RAF

### 5. File structure
- IDM logic in `trafficlab/idm.js` (new file, ~200 lines)
- Tab UI wired in `trafficlab/app.js` (minimal changes — tab init + RAF guard)

## Test Plan
- Open IDM tab, let run at default 120 veh/min: vehicles approach, decelerate, queue, discharge smoothly
- Drag vehicles/min to 600: queue grows without freezing
- Set signal cycle to 10s: rapid phase changes, vehicles react smoothly
- Change lanes 1→6: vehicles redistribute correctly
- Switch between Benchmark and IDM tabs: both resume cleanly
- Open DevTools Performance tab: frame time < 20ms

## Notes
- Reference: Treiber, Hennecke, Helbing (2000) "Congested Traffic States in Empirical Observations and Microscopic Simulations"
- IDM is well-suited here because the intersection has a clear stop line — the signal braking extension (treat red light as stationary virtual vehicle) maps directly.
- The existing MuonCache benchmark simulation does NOT need to change. The IDM tab is for visual demo quality; the benchmark tab remains the correctness/perf comparison tool.
- Warp multiplier on the IDM tab: scale `dt` by warp factor but cap at 3× to avoid vehicles teleporting through the intersection.
