## Goal
Optimize `number_to_value()` in `src/helpers.rs` to use a cheaper integer-
fitness test. The current check does `n.fract() != 0.0` (function call) plus
multiple comparisons. A single `n as i32 as f64 == n` round-trip check is
equivalent and faster.

## Background
`number_to_value()` is called on nearly every arithmetic result in the runtime.
Current implementation:
```rust
if n.is_nan() || n.is_infinite() || n.fract() != 0.0
    || n.abs() > i32::MAX as f64
    || (n == 0.0 && n.is_sign_negative())
{
    js_new_float64(ctx, n)
} else {
    Value::from_int32(n as i32)
}
```

Proposed:
```rust
let i = n as i32;
if i as f64 == n && (n != 0.0 || !n.is_sign_negative()) {
    Value::from_int32(i)
} else {
    js_new_float64(ctx, n)
}
```

The round-trip check `i as f64 == n` handles NaN (NaN != NaN), infinity
(cast saturates, won't match), fractional parts, and range — all in one
FP comparison.
